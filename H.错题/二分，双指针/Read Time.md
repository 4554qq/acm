# CF343C Read Time

## 题目描述

疯狂科学家 Mike 从不用慢速硬盘。他改造的硬盘上不仅有一个，而是有 $n$ 个不同的读写头，可以并行读取数据。

从侧面看，Mike 的硬盘是一个无限的磁道数组。磁道自左至右用正整数编号，从 1 开始。初始时，第 $i$ 个读写头位于编号为 $h_i$ 的磁道上。对于每个读写头，硬盘固件每秒可以将其向左或向右移动一格，或保持在原来的磁道。每个读写头的移动互不影响，读写头之间可以交换相对顺序；同一个磁道上可以有多个读写头。只要有至少一个读写头访问过某个磁道，该磁道就被认为已读取。特别地，所有编号为 $h_1, h_2, \dots, h_n$ 的磁道在操作开始时已经被读取。

Mike 需要读取编号为 $p_1, p_2, \dots, p_m$ 的 $m$ 个不同的磁道。请你确定，最少需要多少秒，硬盘的固件能够移动读写头并读取所有这些指定的磁道。注意，允许读写其它任意数量的磁道。

## 输入格式

输入的第一行包含两个用空格分隔的整数 $n,m$（$1 \leq n,m \leq 10^5$），分别表示读写头的数量以及需要读取的磁道数量。第二行包含 $n$ 个递增的不同整数 $h_i$（$1 \leq h_i \leq 10^{10}$，$h_i < h_{i+1}$），表示每个读写头的初始位置。第三行包含 $m$ 个递增的不同整数 $p_i$（$1 \leq p_i \leq 10^{10}$，$p_i < p_{i+1}$），表示需要读取的磁道编号。

请勿在 C++ 中使用 %lld 格式符读取或输出 64 位整数，建议使用 cin、cout 流或 %I64d 格式符。

## 输出格式

输出一个整数，表示读取所有需要的磁道所需的最短秒数。

## 说明/提示

第一个测试样例与上图一致。在这种情况下，可以用如下方式在 2 秒内完成读取：

1. 第 1 秒，让第 1 个读写头向左移动，并保持在那里；
2. 让第 2 个读写头向左移动两次；
3. 让第 3 个读写头向右移动两次（注意第 6 号磁道一开始已被读取）。

无法在 1 秒内完成全部读取，因为第 3 个头距离第 8 号磁道有 2 格。

# 思路

## 第一版：二分+双指针

时间具有单调性，选择二分答案时间，选出最合适的时间x

所有指针不会互相交叉越位，因为要保持最最优解，

每个磁头都有各个要负责的每片区域，所以我们最极限的时间，每个磁头都要在符合的x中将各自负责的区域扫完，但各自的区域又不知道怎么确定，所以我们用

```
while (r<=m) {
    longlong cost = abs(p[r] -p[l]) +min(abs(p[l] -h[i]),abs(p[r] -h[i]));
    if (cost<=x) r++;
    elsebreak;
    }

```

abs(p[r] - p[l])是磁道的长度,min(abs(p[l] -h[i]),abs(p[r] -h[i])就能确定当前的h[i]磁头选择最近的一段进入

第一次看这里有一个小疑惑，如果真就从右边进了，那后边的如果是分给前边最优的怎么办

其实就是在这个更优的x更多的剩，如果是能去掉当前磁头给右边的话，那当前磁头所花费的时间一定会少，那样子，将这个分配给后边一定会在多负担的情况下给这个也要消耗，**关键就是：如果后边的在更小的x负担不下这个所谓的右边，那一定是不通过的，是需要加大x看当前和后边这个谁能先负担的下，然后后边一直一直是好多个这种“关节”，其性质是一样的**

## 第二版：二分答案 + 双指针贪心

设答案为最小时间 **x**。显然时间具有单调性：

* 如果时间 **x** 可以完成所有覆盖任务
* 那么更大的时间也一定可以完成

因此可以二分答案，再设计 **check(x)** 判断当前时间是否可行。

将磁头位置 **h[]** 和目标位置 **p[]** 排序后考虑。
在最优方案中，每个磁头负责的目标点一定是一段连续区间，并且这些区间按位置顺序依次分配，不会交叉。
因此我们可以贪心地让当前磁头从“当前最左边尚未覆盖的点”开始，尽量向右覆盖更多点。

设当前第 **i** 个磁头要尝试覆盖区间 **p[l..r]**，那么它覆盖这整段区间的最小代价为：

`<span><span>abs(p[r] - p[l]) + min(abs(p[l] - h[i]), abs(p[r] - h[i]))</span></span>`

其中：

* **abs(p[r] - p[l])** 是扫完整段区间的长度
* **min(abs(p[l] - h[i]), abs(p[r] - h[i]))** 表示磁头先走到区间较近的一端再开始扫描

所以在 **check(x)** 中，可以不断扩展右端点：

`<span><span>while (r <= m) {</span> <span>    long long cost = abs(p[r] - p[l]) + min(abs(p[l] - h[i]), abs(p[r] - h[i]));</span> <span>    if (cost <= x) r++;</span> <span>    else break;</span> <span>}</span></span>`

表示当前磁头在时间 **x** 内最多能负责到哪里。

## 为什么这样贪心是对的

这里一个容易产生的疑问是：

如果当前磁头覆盖区间时是“从右边进入”的，那会不会右边这些点本来更适合留给后面的磁头，从而当前这样贪心不最优？

不会。

因为这里的式子只是在计算“当前磁头覆盖整个区间 **[p[l], p[r]]** 的最小代价”，并不表示它真的把右边的点“抢走了”。
只要代价不超过 **x**，就说明这一整段都可以合法地交给当前磁头负责。

而如果某些右边的点不交给当前磁头、改交给后面的磁头：

* 当前磁头的负担确实可能变小
* 但后面的磁头负担一定会变大

如果后面的磁头连这些点都接不住，那么当前时间 **x** 本来就不可行；
如果后面的磁头接得住，那说明在这个 **x** 下本来就存在可行分配，而让当前磁头尽量多覆盖前缀并不会使可行性变差，因为后续磁头面对的只会是一个更靠右的后缀区间，问题性质不变。

因此，在判定可行性时，让每个磁头尽量覆盖当前能覆盖的最长连续前缀是正确的。
